## 二、详细发现（按 CVPR 审稿维度分类）

### 🔴 Reject 级缺陷（必须修复）

#### [R1] 适配器预训练 Epoch 数：论文 50 vs 代码 10
- **CVPR 审稿视角**：Technical Correctness — 训练配置不一致属于基本事实错误
- **问题描述**：论文 Section 4.3 (Implementation Details) 声称适配器经过 "50-epoch training on source-domain data" 预训练为恒等映射。但 `train.py:103` 实际代码为 `for epoch in range(10):`，注释写 "适配器只用 256×256 crop 训练，loss < 0.001 即收敛"。
- **论文依据**：paper.tex L448: "Adapters are pre-initialized to approximate identity mappings via 50-epoch training on source-domain data"
- **代码依据**：`train.py:103`: `for epoch in range(10):  # 适配器只用 256×256 crop 训练，loss < 0.001 即收敛`
- **PAPER_SPEC.md 依据**：L311 写 "10-epoch training"，与代码一致但**与 paper.tex 矛盾**
- **对公平性的影响**：不影响 baseline 对比（adapter 预训练是本方法内部步骤），但属于事实性错误。若 reviewer 复现时发现 10 epoch 而非 50 epoch，会质疑论文的可信度。
- **修复建议**：以代码/PAPER_SPEC 为准，将 paper.tex L448 的 "50-epoch" 改为 "10-epoch"（或确认实际运行时是否用了 50，若是则更新代码）。
- **预计修复工作量**：5 分钟

#### [R2] 论文报告 3-seed mean±std 但所有表格无标准差
- **CVPR 审稿视角**：Reproducibility — CVPR 审稿指南要求报告统计显著性
- **问题描述**：论文 Section 4.3 明确写 "Random seeds: 3 runs (42, 123, 2026), report mean ± std"，但 Table 1-5 中**所有数字均无标准差**。+0.87 dB (Full MoTA vs frozen) 和 T=10 vs T=15 (22.72 vs 22.36) 的差异是否统计显著完全无法判断。
- **论文依据**：paper.tex L441: "Random seeds: 3 runs (42, 123, 2026), report mean ± std"
- **代码依据**：`eval.py:209` 设置 `torch.backends.cudnn.deterministic = True`，有 `--n_runs` 参数但当前所有实验均用 `--n_runs 1`；`run_multi_seed.sh` 存在于 CLAUDE.md 中但结果未纳入论文。
- **修复建议**：(a) 运行 3-seed 评测并更新所有表格加 ±std；或 (b) 删除 Section 4.3 中 "report mean ± std" 的声明（不推荐，会降低可信度）。
- **预计修复工作量**：2-3 小时（运行 3-seed × 多组实验 + 更新 LaTeX）

### 🟡 Major Concern（审稿人会要求澄清）

#### [M1] "Architecture-Flexible" 声明仅 1 个 backbone 有 full TTA 结果
- **问题描述**：论文标题和全文中使用 "architecture-flexible"，但仅有 WDNet (wavelet-based) 完成了 full TTA 评测。DDA (CNN-based, MBCNN) 仅报 frozen baseline，原因为 "internal downsampling layers produce weak gradients"。这意味着架构灵活性仅在 1 个 backbone 上被验证。
- **论文依据**：Title + Abstract + Section 1 Contributions 声称 "architecture-flexible"
- **代码依据**：`eval.py:256-269` — DDA 加载路径存在，adapter 可插入，但因架构限制 (4× 下采样导致梯度断连) 无法有效 TTA
- **建议**：(a) 补充至少 1 个非 wavelet backbone 的 full MoTA 结果（如 ESDNet-lite 或简版 ResNet-based demoiréing model）；或 (b) 在 Limitations 中明确此限制，并限定声明为 "demonstrated on wavelet-based and CNN backbones with varying degrees of TTA compatibility"
- **严重度**：中高 — 若 reviewer 抓住此点，"architecture-flexible" 的核心 claim 缺乏证据支撑

#### [M2] MFD 3 通道语义映射未在论文中说明
- **问题描述**：MFD 输出 3 通道概率图，但 MASS 仅使用 channel 0 (LH) 和 channel 1 (HL)。Channel 2 (HH) 在训练中被优化但在推理时完全未使用。
- **论文依据**：Section 3.4.3 MASS 公式显示仅 LH 和 HL 被衰减，但未说明 MFD 输出的 3 通道具体语义
- **代码依据**：`mass.py:52-55`:
  ```python
  # channel 0 → LH, channel 1 → HL, channel 2 → HH (trained but unused at inference)
  LH_attenuated = LH * (1.0 - alpha * M_ds[:, 0:1, :, :])
  HL_attenuated = HL * (1.0 - alpha * M_ds[:, 1:2, :, :])
  ```
- **建议**：在 Section 3.4.2 MFD 输出描述中加一句话说明 3 通道分别对应 LH/HL/HH 的摩尔纹概率，HH 通道在训练中作为辅助任务但推理时不用于衰减（因 HH 主要含高频纹理而非摩尔纹）。

#### [M3] 跨域消融仅在 100-image subset 上进行
- **问题描述**：Table `tab:cd_diagnostics` 的跨域消融 (w/o MASS, w/o L_reg) 仅基于 TIP2018 的 100 张子集。全量 CD 评测 (11851 张) 下 MASS/L_reg 的关键性是否仍然成立未知。且 100-image subset 上 frozen PSNR (20.33 dB) 与全量 (18.91 dB) 差异达 1.42 dB，子集可能不具代表性。
- **论文依据**：Table `tab:cd_diagnostics` footnote: "CD frozen PSNR on the 100-image diagnostic subset (20.33 dB) differs from the full TIP2018 result (18.91 dB)"
- **代码依据**：`diagnose_cd.py:245-322` — `diagnose_cd_ablation()` 默认 `n_images=100`
- **建议**：(a) 扩大 CD 消融至至少 500 张以减少采样偏差；或 (b) 在论文中更明确地标注子集大小和代表性问题

#### [M4] FDA `diag(AB^T)` 公式与代码实现的语义差异
- **问题描述**：论文公式写 `δ = diag(AB^T)`，暗示矩阵乘法。但代码实际是 `(self.A * self.B).sum(dim=1)`，即逐元素乘积 (Hadamard product) 后沿 rank 维求和。两者对**对角元素**数学等价（`diag(A @ B^T)_c = Σ_k A_{c,k} · B_{c,k}`），但 `diag(AB^T)` 的标准读法会让人以为需要 O(C²r) 的矩阵乘 — 实际代码只需 O(Cr) 的逐元素乘。
- **论文依据**：paper.tex L218-219: `δ = diag(A B^T) ∈ R^C, A, B ∈ R^{C×r}`
- **代码依据**：`adapters.py:43`: `delta_freq = (self.A * self.B).sum(dim=1)`
- **建议**：将公式改为 `δ_c = Σ_{k=1}^{r} A_{c,k} B_{c,k}` 以精确对应代码实现。当前 paper.tex L223 已有文字说明 "The per-channel modulation δ_c = Σ_{k=1}^{r} A_{c,k} B_{c,k}"，公式与文字存在细微不一致。

#### [M5] 适配器自动检测逻辑依赖硬编码层名过滤
- **问题描述**：`insert_mota_adapters()` 通过扫描 Conv2d 层并过滤名称含 `dec.`, `rec.`, `final`, `conv_last` 的层来自动选择插入点。这种基于字符串匹配的策略对 backbone 命名约定高度敏感。若用户换了 backbone（如 ESDNet、MoiréNet），适配器可能插入到错误位置。
- **代码依据**：`adapters.py:109-113`:
  ```python
  if any(skip in name for skip in ('dec.', 'rec.', 'final', 'conv_last')):
      continue
  ```
- **建议**：这是工程实现层面的问题，不影响论文核心 claim，但建议在代码文档中说明此假设，或在 `insert_mota_adapters` 中提供显式的 `channel_list` 参数覆盖自动检测。

### 🟢 Minor Issue

#### [m1] FDA-only/SGA-only 的 SSIM 标记为 "---" 但代码已计算
- **问题描述**：论文 Table 3 (Ablation) 中 FDA-only (24.12 dB) 和 SGA-only (23.56 dB) 的 SSIM 列标记为 "---"，注释说 "SSIM and LPIPS for FDA-only were not computed in the component ablation"。但 `eval.py:45-68` 的 `compute_metrics()` 在 SSIM 可用时**始终计算 SSIM**，不存在 "未计算" 的情况。
- **代码依据**：`eval.py:51-57` — SSIM 在 `_SSIM_AVAILABLE` 时总是被计算
- **建议**：如果数据实际存在，补填 SSIM 值（预计 FDA-only SSIM 应高于 Full MoTA 的 0.4852）

#### [m2] MASS alpha 参数声称范围 [0.3, 0.7] 但无 sensitivity analysis
- **问题描述**：论文声称 α ∈ [0.3, 0.7]，但所有实验固定 α=0.5，未提供 α 敏感度分析
- **建议**：可接受（消融实验已充分覆盖其他组件），但建议在 Limitation 中提一句

#### [m3] SSIM 使用 skimage 而非 PyTorch 原生实现
- **问题描述**：`eval.py:57` 使用 `skimage.metrics.structural_similarity` 计算 SSIM (data_range=255, channel_axis=2)。这与其他 demoiréing 论文常用实现可能不同（部分工作自写 differentiable SSIM），可能导致不可直接对比
- **建议**：可接受，但建议在论文中注明 SSIM 实现来源

#### [m4] 部分变量命名不一致
- `representational competition` → `representation competition` (paper.tex 多处使用前者)
- paper.tex L123: "Moiré Frequency Detector" vs paper.tex L169: "Moiré Frequency Detector (MFD)" — 首次出现时应给全称+缩写

#### [m5] DDA CD PSNR=12.26 dB 低于 bicubic 退化水平
- **问题描述**：12.26 dB 的 PSNR 低于直接对输入做 bicubic 下采样+上采样的水平(~18-20 dB)，论文解释为 "internal 4× downsampling layers" 导致。但未验证：DDA 在 TIP2018 全分辨率 (约 400×400) 上的输出尺寸实际是多少？如果输出是 100×100 而非 400×400，PSNR 的计算基准就有问题。
- **代码依据**：DDA MBCNN 内部 4× 下采样 (CLAUDE.md L75 确认)
- **建议**：加 footnote 说明 DDA 输出分辨率和 PSNR 计算时的上采样方式

## 八、审查结论（模拟 CVPR 审稿意见风格）

```
This paper presents MoTA, an architecture-flexible test-time adaptation
framework for image demoiréing. The core idea — using frequency-selective
self-supervision (MASS) to guide lightweight adapter optimization at test
time — is well-motivated and the MASS design (exploiting moiré's mid-frequency
energy concentration) is genuinely clever. The paper has improved substantially
since prior revisions: cross-domain diagnostic experiments now provide
quantitative evidence for the failure modes, the FDA-only recommendation is
supported by ablation data, and the claims-evidence alignment is much stronger.

My main concern is about factual accuracy in implementation reporting:
the paper claims 50-epoch adapter pre-training while the code implements
10 epochs (train.py:103). While this does not affect the comparative results,
it erodes trust in the reported experimental protocol. Additionally, the
paper claims 3-seed evaluation with mean±std (Sec 4.3) but no standard
deviations appear in any table — without them, the statistical significance
of the +0.87 dB gain and the T=10 vs T=15 difference cannot be assessed.

The "architecture-flexible" claim, while technically correct (the adapter
design is architecture-agnostic), is empirically supported by only one
backbone with full TTA evaluation. A second backbone with complete MoTA
results would substantially strengthen this claim.

On balance, the MASS insight, honest reporting of negative results, and
thorough diagnostic analysis constitute a meaningful contribution to the
TTA-for-restoration literature. If the authors address the factual
discrepancies (adapter pre-training epochs, missing std values) in the
rebuttal, my score would increase to Weak Accept.
```
