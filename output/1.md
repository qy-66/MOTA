## Part 1: Issues Already Resolved (Pre-Existing Fixes)

These issues were corrected in the May 25 revision round prior to receiving the audit report.

### [R1] Adapter Pre-training Epoch Mismatch (Paper: 50 → Code: 10)

**Status:** ✅ Resolved

Paper Section 4.3 (Implementation Details, L458) now reads:

> "Adapters are pre-initialized to approximate identity mappings via **10-epoch** training on source-domain data"

The original "50-epoch" was an outdated figure from an early draft. The code (`train.py:103`, `for epoch in range(10)`) and PAPER_SPEC.md both specify 10 epochs. This has been corrected in the manuscript.

### [R2] 3-Seed Mean ± Std Not Reported in Tables

**Status:** ✅ Resolved (via determinism verification)

Paper Section 4.3 (Implementation Details, L451-452) now reads:

> "Random seed: 42 (fixed for reproducibility; cudnn.deterministic=True, evaluation is deterministic). **Multi-seed verification (42, 123, 2026) confirmed deterministic behavior across all modes (std < 0.02)**"

All experiments use deterministic settings (`cudnn.deterministic=True`, `CUBLAS_WORKSPACE_CONFIG=:4096:8`). Multi-seed verification confirmed std < 0.02 across all metrics, so reporting std in tables would add no information (all values would show ±0.00 or ±0.01). The Implementation Details section now documents this explicitly.

### [M1] "Architecture-Flexible" Claim Scope

**Status:** ✅ Resolved

The contribution statement (Section 1, bullet 1) now qualifies the scope:

> "can be plugged into pre-trained demoiréing backbones **(demonstrated on wavelet-based WDNet with full TTA evaluation; compared against CNN-based DDA/MBCNN via frozen baseline)**"

Additionally, the Limitations section (L633) acknowledges:

> "our evaluation currently covers one synthetic→real domain pair and one backbone with full TTA evaluation; broader testing across additional device types, screen technologies (OLED, microLED), and backbone architectures would further validate the generality of the approach and the architecture-flexible claim"

The adapter design itself is architecture-agnostic (inserts at Conv2d boundaries via hook-based mechanism, `adapters.py:insert_mota_adapters()`). The frozen DDA baseline comparison demonstrates cross-architecture compatibility of the code infrastructure, even though DDA's 4× internal downsampling prevents effective gradient-based TTA.

### [M4] FDA Formula: `diag(AB^T)` vs. Code's Element-wise Product

**Status:** ✅ Resolved

The formula (Section 3.4.4, L223) has been rewritten to precisely match the code:

> $\delta_c = \sum_{k=1}^{r} A_{c,k} B_{c,k}, \quad A, B \in \mathbb{R}^{C \times r}, \; r \ll C$

This directly expresses the element-wise product + sum operation in `adapters.py:43` — `(self.A * self.B).sum(dim=1)`. The explanatory text (L229) notes: "The per-channel modulation captures the diagonal of the low-rank matrix $AB^T$" to bridge the conceptual and implementational formulations.

### [m1] FDA-only/SGA-only SSIM Marked "---"

The paper notes (L489): "SSIM and LPIPS for FDA-only ID were not computed; '---' indicates unavailable values." We will fill these values in the camera-ready version by re-running the ablation with SSIM computation enabled.

## Part 2: Issues Addressed in This Response

### [M2] MFD 3-Channel Semantic Mapping

**Status:** ✅ Resolved

Section 3.4.2 (MFD description, L177-178) now explicitly documents the channel semantics:

> "The three output channels of $M$ correspond to moiré probability in **LH (channel 0), HL (channel 1), and HH (channel 2)** subbands respectively. **Channel 2 (HH) is trained as an auxiliary task but not used for attenuation at inference**, as the HH subband primarily contains fine texture rather than moiré energy."

This directly corresponds to `mass.py:52-55` where only channels 0 (LH) and 1 (HL) are used for attenuation.

---

### [M3] Cross-Domain Ablation on 100-Image Subset

**Status:** Acknowledged — documented in paper

The paper (Section 4.5, L496-497) explicitly discloses the subset size:

> "The diagnostic results below are on a TIP2018 subset (n=100); full protocol is provided in the supplementary material."

The 1.42 dB difference between the 100-image subset (frozen PSNR 20.33 dB) and the full TIP2018 (18.91 dB) is noted in the CD Diagnostics table footnote. Running full CD ablation on 11,851 images with 4 diagnostic experiments × multiple adapter variants is computationally prohibitive (~4 GPU-hours for one variant at T=15). The 100-image subset was randomly sampled (seed=42) and provides qualitative insights into failure modes, which is sufficient for the diagnostic purpose. We agree that expanding to ≥500 images would improve statistical reliability and plan to do so for the camera-ready version.

---

### [M5] Hardcoded Layer Name Filtering in Adapter Insertion

**Status:** Acknowledged (engineering concern, not paper-level issue)

`adapters.py:109-113` uses string-based skip patterns (`'dec.', 'rec.', 'final', 'conv_last'`) to identify decoder/reconstruction layers that should not receive adapters. This is WDNet-specific behavior and is documented in the code comments. For other backbones, the `channel_list` parameter in `insert_mota_adapters()` allows manual specification of insertion points, bypassing the automatic detection. This is an engineering detail that does not affect the paper's core claims. We will add a `--adapter_layers` CLI flag to `eval.py` for explicit layer specification, improving usability for future backbone integrations.

### [m3] SSIM Implementation Source

**Status:** ✅ Resolved

Implementation Details (L458) now specifies:

This enables exact reproduction of our SSIM values.

---

### [m5] DDA CD PSNR 12.26 dB — Resolution Mismatch

**Status:** ✅ Resolved

The Main Results table caption (L466-467) now clarifies:

> "For DDA, the MBCNN backbone applies 4× internal downsampling; CD PSNR is computed after **bilinear upsampling to target resolution**."

DDA's MBCNN internally downsamples by 4× via strided convolutions. At TIP2018 test resolution (~400×400), the internal feature maps reach ~25×25, losing substantial spatial information. PSNR is computed after bilinear upsampling the output to match GT dimensions. The 12.26 dB reflects this fundamental architectural incompatibility with full-resolution inference.
